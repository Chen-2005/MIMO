## 二、风格控制增强 ✅ 已完成

### 2.1 内联音频标签 ✅

前端 `TextInput.tsx` 已实现标签插入面板：

- 点击"插入表现力标签"展开面板
- 标签分组：情绪(12)、基调(8)、音色(6)、方言(4)、角色(5)、特殊(唱歌)
- 点击标签在光标位置插入 `(tag)`
- 后端无需改动，标签直接在文本中传给 MiMo API

### 2.2 导演模式 ✅

前端 `StyleControls.tsx` 已实现导演模式面板：

- 三个结构化输入：[角色]、[场景]、[指令]
- 每个维度提供 4 个预设提示，点击填入
- 格式化为 `style_prompt` 传给后端
- 后端映射为 MiMo API 的 `user` 消息

### 2.3 自然语言控制 ✅

`style_prompt` 文本框支持自由文本描述，前端提供 3 个风格预设按钮（自然讲解、温和叙述、短视频旁白）。

## 三、长文本分段策略修正 ✅ 已完成

**改动**：`backend/app/infra/text_segmentation.py` 默认 `max_chars` 从 500 改为 2500。

**分段策略**：
1. 按句子边界（`。！？!?.\n`）切分
2. 贪心合并，每段不超过 2500 字
3. 单句超长时按逗号/分号切分
4. 最终兜底按字符数强制切分

## 四、Voice Design 描述引导 ✅ 已完成

前端 `VoiceDesignTab.tsx` 已实现：

**描述生成器**：
- 性别选择（女/男）
- 年龄段选择（少女/青年/中年/成熟）
- 声音质感选择（4 种）
- 语速选择（4 种）
- 情绪基调选择（5 种）
- 补充特征输入
- 自动生成描述文本

**快速模板**：
- 温柔知性女声
- 活力少年男声
- 沉稳播报男声
- 甜美少女

**提示文字**：好的描述 = 年龄性别 + 声音质感 + 语速节奏 + 情绪基调。

## 六、API 调用优化 ⏭️ 已决定跳过

**决定**：保持 httpx 直接调用，不迁移到 OpenAI SDK。

**原因**：
- 当前 httpx 实现已稳定
- 两者本质都是 HTTP 调用，只是客户端不同
- 迁移有兼容性风险，收益不大
- Skill 文档本身也推荐 API 方式

## 七、其他可选增强

### 7.1 唱歌模式 ✅ 已完成

前端标签面板已包含 `(唱歌)` 标签（特殊分组，橙色样式），用户点击即可插入。

### 7.2 方言支持 ✅ 已完成

前端标签面板已包含：`(东北话)` `(四川话)` `(河南话)` `(粤语)`。

不在原始 Skill 优化清单中，但已实现：
- 试听后可选择发布/取消发布
- `is_public` 控制可见性
- 发布后团队所有成员可见
